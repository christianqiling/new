# 店内计时收费系统

一个**零依赖**的网页版店内计时收费系统：顾客自助进店、按进店时长（分时段单价）实时扣费，管理员管理用户、定价与营业数据。全部基于 Node.js 内置模块（`node:http` + `node:sqlite` + `node:crypto`），**无需 `npm install`**，一条命令即可运行。浅色简洁界面。

## 功能

**顾客**
- 自助注册（用户名 + QQ 号 + 密码）、登录
- 进店 / 离店，按当前时段单价**实时扣费**，余额可扣为负（离店后红色显示），余额 ≤ 0 时不能进店
- 实时查看余额、本次时长与费用、价格表、**店内公告**
- 查看店内在店人数与人员（顾客姓名以首字 + ** 打码，管理员姓名全员可见，均显示头像）
- **我的信息**：修改用户名、昵称、上传头像（默认头像为用户名首字）、修改密码
- 我的充值 / 消费明细

**管理员（管理员 / 超级管理员）**
- 进店 / 离店打卡（不计费），可查看在店全部真实姓名
- 强制结束某顾客会话
- **用户列表**（合并）：新增 / 删除用户、修改余额 / QQ / 用户名 / 昵称、恢复默认头像、**一键导出 Excel(.xlsx)**；超级管理员可设置用户身份与权限。修改 / 删除用户信息需输入**操作密码**
- **营业管理**（合并）：营业报表（当日营业额、进出记录、近 7 天柱状图）、分时段营业定价、流水明细（按用户 / 类型筛选）
- **店内公告**：在主页发布公告，可置顶 / 删除
- 超级管理员：分配管理员及其权限、修改操作密码

> 角色名称：超级管理员（最高权限） / 管理员（分配权限） / 用户（顾客）。

## 新增能力（近期）

- **免费额度**：管理员可单独或"统一发放"免费额度，游玩时优先扣除免费额度，再扣余额（余额可为负，离店后红色显示，可用额度≤0 时不可进店）。
- **管理员也计费**：所有管理员同样有"我的账户"并按时长计费；超级管理员可在用户列表设置"管理员优惠"百分比。
- **扫码充值**：每位用户"我的账户"可生成充值二维码（每 10 秒刷新）；管理员首页"用户充值"用扫码枪扫描即可为其充值。
- **会员卡片**：每人至多 3 张卡片（卡号由读卡器录入）；用户可删除自己的卡片，管理员可增删用户卡片。
- **店内公告**：主页公告板，管理员可发布 / 置顶 / 删除。
- **在店折叠列表**：在店顾客 / 管理员分别为可展开的折叠列表，显示头像与姓名。
- **移动端适配 + 浅色主题（主题色 #D7EEFF）**，页面切换带轻量动画。

## 运行

```bash
./start.sh            # 或 PORT=8080 ./start.sh
# Windows: node src/server.js
```

浏览器打开 `http://localhost:3000`。

> 本项目用到 Node 内置 SQLite，**需要 Node.js ≥ 22.5**。
> 沙箱环境若存在无效的 `NODE_OPTIONS` 预加载，请用：`env -u NODE_OPTIONS node src/server.js`（`start.sh` 已自动处理）。

## 默认账号与密码（请尽快修改）

| 项目 | 默认值 | 说明 |
|------|--------|------|
| 超级管理员 | `admin` / `admin123` | 首次启动自动创建 |
| 操作密码 | `123456` | 用户列表中修改 / 删除用户等敏感操作需输入；可在"用户列表 → 设置操作密码"修改 |
| 默认时价 | 全天 10 元/小时 | 可在"营业管理 → 营业定价"修改 |

## 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `3000` | 服务端口 |
| `DB_PATH` | `data/store.db` | SQLite 数据库文件路径 |
| `TICK_MS` | `60000` | 实时扣费结算间隔（毫秒） |
| `SUPER_ADMIN_USER` / `SUPER_ADMIN_PASS` | `admin` / `admin123` | 初始超级管理员 |
| `OP_PASSWORD` | `123456` | 初始操作密码 |
| `ENABLE_MOCK_PAY` | 关 | 设为 `1` 开启演示用模拟在线支付（默认关闭，普通用户不可自助在线充值） |
| `WECHAT_MCH_ID` / `ALIPAY_APP_ID` | 关 | 配置后启用对应在线支付渠道 |

## 目录结构

```
src/
  server.js    HTTP 服务器、路由、静态资源、扣费定时器
  api.js       各接口处理逻辑
  db.js        SQLite 初始化、建表、种子数据
  auth.js      密码哈希、会话、角色与权限
  billing.js   分时段计费引擎（实时扣费、余额耗尽结算）
  payment.js   可插拔在线支付框架（mock / 微信 / 支付宝）
public/
  index.html / app.js / styles.css   纯原生前端单页应用（浅色主题）
data/
  store.db     运行时自动生成的数据库
```

## 计费与隐私说明

- 货币以"分"为单位存储，避免浮点误差，展示时换算为元。
- 进店要求余额 > 0；游玩中每 `TICK_MS` 结算一次并写入流水；前端每秒插值让数字实时跳动。
- 跨时段游玩按各时段费率分段计费；未覆盖的时段视为免费；余额耗尽时在耗尽时刻自动结束。
- 顾客之间互不显示真实姓名（首字 + ** 打码），管理员可见全部全名。
- 在线支付为可插拔框架：默认仅保留接口，配置微信/支付宝商户凭证并联网后即可启用，到账由支付平台回调触发（已实现幂等入账）。
